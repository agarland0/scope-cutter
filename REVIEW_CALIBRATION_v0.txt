REVIEW_CALIBRATION_v0 — EXISTENCE

Purpose
Calibrate the Existence invariant using borderline phase examples.
This is proof that Existence judgments are stable enough to encode.

Method
- Use PHASE_TEMPLATE_v0.txt.
- For each candidate phase, record independent verdicts:
  - CTO
  - Senior Engineer
  - Security
- Resolve disagreements by tightening wording in the phase or Existence definition,
  not by adding new lenses.

Verdict Legend
PASS = Existence satisfied (single standalone artifact/behavior)
BLOCK = Existence not satisfied
