THREAT MODEL (lightweight)

Assumptions
- Inputs may contain sensitive data (internal docs, customer info, proprietary text, metrics, logs).
- Tooling is local-first and must not transmit or retain data.

Non-negotiables
- No network access (no HTTP calls, telemetry, or remote APIs).
- No persistent storage of user inputs beyond the file the user provides.
- No logging of raw input rows or sensitive values by default.
- No background processes or daemons.

Primary risks
- Accidental data leakage via console logs, debug files, or crash dumps.
- Introducing dependencies that add telemetry or remote calls.
- Writing temporary files or caches containing user data.

Mitigations (v0+)
- Keep output minimal and avoid printing raw data.
- Prefer standard library; review any new dependency.
- Security review required for any change touching IO, logging, or dependencies.
