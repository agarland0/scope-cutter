ACTIVE PROJECT CONTEXT — SCOPE CUTTER

Status
This project is in definition phase.
No implementation decisions are locked unless explicitly stated here.

Purpose
Scope Cutter exists to evaluate whether a proposed release phase
legitimately exists as a standalone thing.

It does not optimize, analyze, recommend, or improve ideas.
It determines legitimacy and blocks illegitimate releases.

Primary Invariant (v0)
Existence:
A release phase must ship a single, standalone artifact or behavior
that delivers value on its own, without reliance on future phases.

If this invariant is not satisfied, the phase is illegitimate.

Authority
Scope Cutter is a legitimacy gate, not a collaborator.
It is allowed to block.
It is not required to help the user fix what is blocked.

Out of Scope (explicit)
- Product ideation or validation
- Metrics, dashboards, or analysis
- Recommendations or suggestions
- Multi-phase planning
- Roadmap design
- Salience, trust, or reversibility lenses
- UX, UI, or distribution concerns

Allowed Work Right Now
- Tightening definitions of “artifact”, “behavior”, and “standalone”
- Defining false positives / false negatives acceptable in v0
- Clarifying what Existence explicitly does NOT cover
- Designing review and decision boundaries
- No code unless explicitly authorized in this file

Interaction with Other Systems
- Forge may generate candidate artifacts, but has no authority to declare legitimacy.
- Review roles are applied using the canonical review system.
- Project-local copies of the review system are mirrors only.

Non-Goals
This project does not need to:
- Feel helpful
- Feel encouraging
- Justify its existence with breadth
- Solve adjacent problems

Done for v0 (definition phase)
The definition phase is complete when:
- The Existence invariant is unambiguous
- Reasonable reviewers agree on what would be blocked vs allowed
- No additional lenses are required to make the judgment meaningful

Notes

If there is a conflict between this file and other documents,
this file wins.
